### 11.2 总体设计

#### 11.2.1 需求规定

本软件的主要功能需求已在SRS第3.1节中详细规定，简要概括如下：

| 功能模块 | 功能描述 |
| :--- | :--- |
| 图片取色 | 上传图片，通过K-Means聚类算法提取3～8个主色，生成配色方案 |
| 色轮规则生成 | 基于基色和色彩和谐规则（互补色、邻近色、三角色、矩形色、单色渐变）生成衍生色 |
| 随机创意生成 | 在设定的饱和度/明度约束范围内随机生成配色，支持颜色锁定机制 |

| 历史管理 | 保存、回看、删除配色历史记录（最多50条） |

#### 11.2.3 基本设计概念和处理流程

**11.2.3.1 基本设计概念**

本软件采用**前端单体应用（SPA）架构**，所有功能在浏览器端完成，无需后端服务器。前端遵循**MVVM设计模式**，通过Vue 3的响应式数据绑定实现UI状态管理。系统的核心设计理念包括：

（1）**模块化设计**：将配色生成、色彩计算、AI交互、UI预览等功能拆分为独立模块，各模块通过事件总线或状态管理进行通信，降低耦合度。

（2）**算法与服务分离**：色彩计算核心算法（HEX/RGB/HSL转换、色轮规则计算、K-Means聚类）封装为独立的工具库，与UI逻辑分离，便于测试和维护。

（3）**插件化导出架构**：导出功能采用适配器模式，为不同格式（ASE、CSS、PNG）提供统一的导出接口，便于后续扩展新的导出格式。

（4）**LLM服务可插拔**：AI主题化生成模块支持多种LLM后端（本地Ollama、云端通义千问/DeepSeek API），通过配置文件切换，降低外部依赖风险。

（5）**离线优先策略**：除AI生成功能外，所有核心功能均可在无网络环境下正常运行，历史数据通过IndexedDB本地持久化。

##### 11.2.4.3 模块间控制与被控制关系

- `App` 为顶层控制组件，持有全局配色方案状态，通过事件总线或Props向下传递数据给各子组件。
- 第2层各组件接收`App`传递的状态，处理各自领域的交互，并将用户操作事件向上传递或调用工具层服务。
- 第3层子组件隶属于第2层特定组件（如`ImageUploader`隶属于图片取色功能入口，`RuleSelector`和`SlidersPanel`隶属于配色调整区域）。
- 第4层工具/服务层为所有上层组件提供公用能力，不持有UI状态，仅提供函数式API。

### 11.4 运行设计

#### 11.4.1 运行模块组合

系统在不同的用户操作场景下，会组合不同的模块运行：

| 运行场景 | 运行的模块组合 |
| :--- | :--- |
| 图片取色流程 | ImageUploader → ImageExtractor → App → PaletteDisplay → PreviewPanel → StorageService |
| 色轮规则生成流程 | ColorWheel → RuleSelector → ColorUtils → App → PaletteDisplay → PreviewPanel → StorageService |
| 随机创意生成流程 | App（随机生成入口）→ ColorUtils → App → PaletteDisplay → PreviewPanel → StorageService |
| AI主题化生成流程 | AIGenerator → LLMService → App → PaletteDisplay → PreviewPanel → StorageService |
| 色板微调流程 | PaletteDisplay → SlidersPanel → ColorUtils → App → PaletteDisplay → PreviewPanel |
| 导出流程 | ExportPanel → ExportService → 浏览器下载 |
| 历史回看流程 | HistoryPanel → StorageService → App → PaletteDisplay → PreviewPanel |
| 页面初始化流程 | App → StorageService → App → PaletteDisplay → PreviewPanel |

#### 11.4.2 运行控制

各运行场景的控制方式方法和操作步骤：

**（1）图片取色运行控制：**
1. 用户点击上传按钮或拖拽图片到上传区域；
2. `ImageUploader`校验文件格式与大小，显示缩略图预览；
3. 用户调整提取颜色数量滑块（3～8个），点击“提取主色”按钮；
4. `ImageUploader`调用`ImageExtractor.extractColors()`，传入图片文件和数量参数；
5. `ImageExtractor`显示“处理中”加载状态，执行K-Means聚类计算（≤3s）；
6. 提取完成后，返回主色数组，触发`palette:updated`事件；
7. `App`更新全局配色方案状态，`PaletteDisplay`刷新色板，`PreviewPanel`刷新预览；
8. `StorageService`自动保存配色到历史记录。

**（2）色轮规则生成运行控制：**
1. 用户在`ColorWheel`上拖拽选择基色；
2. `ColorWheel`实时计算色相角度，更新基色HEX值并显示；
3. 用户在`RuleSelector`下拉列表中选择色彩和谐规则；
4. `RuleSelector`调用`ColorUtils.generateHarmony(baseHex, rule)`；
5. `ColorUtils`基于HSL色彩空间计算衍生色，返回配色数组；
6. 触发`palette:updated`事件，后续流程同上。

**（3）AI主题化生成运行控制：**
1. 用户在`AIGenerator`文本框中输入风格描述（如“赛博朋克霓虹紫”）；
2. 用户点击“AI生成”按钮；
3. `AIGenerator`调用`LLMService.generateThemeFromText(text)`；
4. `LLMService`显示等待状态提示，构造Prompt，发送HTTP POST请求；
5. 收到响应后解析JSON，若解析失败则回退到色轮规则生成；
6. 返回配色数组，触发`palette:updated`事件，后续流程同上。

**（4）导出运行控制：**
1. 用户点击“导出”按钮，展开格式选择下拉列表；
2. 用户选择导出格式（ASE/CSS/PNG）；
3. `ExportPanel`调用对应的`ExportService`方法，传入当前配色方案；
4. `ExportService`根据格式规范生成文件内容（ASE为二进制Blob，CSS为文本Blob，PNG为Canvas导出的图片Blob）；
5. 创建下载链接，触发浏览器下载。

**（5）快捷键运行控制：**
- 空格键：调用`ColorUtils.randomColors()`生成随机配色，触发`palette:updated`事件；
- Ctrl/Cmd+C：若当前选中某个色块，将对应HEX值写入剪贴板。

#### 11.4.3 运行时间

各运行模块组合占用资源的时间估计：

| 运行模块组合 | 预计占用CPU时间 | 预计内存占用 |
| :--- | :--- | :--- |
| 图片取色流程 | 主色提取≤3s（含K-Means计算），其他≤100ms | 图片加载后约占用图片尺寸×4字节内存 |

| 导出流程 | ≤500ms（PNG导出可能稍长） | 约10MB（生成Blob时） |

（6）**数据迁移机制**：`StorageService`预留数据版本号字段，当历史记录数据结构发生变更时，可实现旧版本数据的自动迁移。

（7）**错误上报（可选）**：后续版本可接入Sentry等前端错误监控服务，实现生产环境错误自动上报。

#### 12.3.6 算法

（1）**HEX与RGB转换算法**：
```
hexToRgb(hex):
  hex = hex.replace(/^#/, '')
  r = parseInt(hex.substring(0,2), 16)
  g = parseInt(hex.substring(2,4), 16)
  b = parseInt(hex.substring(4,6), 16)
  return {r, g, b}

rgbToHex(r, g, b):
  return '#' + [r, g, b].map(x => x.toString(16).padStart(2, '0')).join('')
```

（2）**RGB与HSL转换算法**：
```
rgbToHsl(r, g, b):
  r /= 255, g /= 255, b /= 255
  max = Math.max(r, g, b), min = Math.min(r, g, b)
  l = (max + min) / 2
  
  if max == min:
    h = s = 0
  else:
    d = max - min
    s = l > 0.5 ? d / (2 - max - min) : d / (max + min)
    switch max:
      case r: h = (g - b) / d + (g < b ? 6 : 0)
      case g: h = (b - r) / d + 2
      case b: h = (r - g) / d + 4
    h /= 6
  
  return {h: h * 360, s: s * 100, l: l * 100}
```

（3）**色轮规则计算算法**：
```
generateHarmony(baseHex, rule):
  baseHsl = hexToHsl(baseHex)
  angles = getRuleAngles(rule)  // 例如互补色: [180]
  colors = [baseHex]
  for angle in angles:
    newHue = (baseHsl.h + angle) % 360
    newHsl = {h: newHue, s: baseHsl.s, l: baseHsl.l}
    colors.push(hslToHex(newHsl))
  return colors
```

各规则的色相偏移角度：
- 互补色（complementary）：`[180]` → 2个颜色
- 邻近色（analogous）：`[30, -30]` → 3个颜色
- 三角色（triadic）：`[120, 240]` → 3个颜色
- 矩形色（tetradic）：`[60, 180, 240]` → 4个颜色
- 单色渐变（monochromatic）：`[0, 0, 0, 0]` + 不同S/L → 5个颜色

### 12.4 程序2（ImageExtractor）设计说明

#### 12.4.1 程序描述

`ImageExtractor`为图片主色提取模块，负责从上传的图片文件中提取主色调。该模块使用K-Means聚类算法对图片像素进行聚类，返回按面积占比排序的主色列表。该模块为独立工具模块，根据调用加载执行。

#### 12.4.2 功能

| 函数名 | 输入 | 处理 | 输出 |
| :--- | :--- | :--- | :--- |
| `extractColors(imageFile, count)` | 图片File对象，提取颜色数量（3～8） | 读取图片→缩放→像素采样→K-Means聚类→排序 | 主色HEX数组（长度=count） |
| `preprocessImage(img, maxWidth)` | Image对象，最大宽度 | 缩放图片到合理尺寸，绘制到Canvas，获取像素数据 | ImageData对象 |

#### 12.4.3 性能

- 提取时间要求：≤3s（含图片读取和K-Means计算）；
- 图片缩放策略：最大宽度800px，降低像素处理量；
- 颜色数量精度：提取的颜色数量与指定数量误差≤±1。

#### 12.4.4 输入项

| 输入项 | 标识 | 类型 | 格式/范围 |
| :--- | :--- | :--- | :--- |
| 图片文件 | `imageFile` | File | JPG/PNG/WebP，≤10MB |
| 提取数量 | `count` | number | 3～8 |
| 最大宽度（预处理） | `maxWidth` | number | 默认800 |

#### 12.4.5 输出项

| 输出项 | 标识 | 类型 | 格式/范围 |
| :--- | :--- | :--- | :--- |
| 主色数组 | `colors` | string[] | 长度3～8的HEX字符串数组 |

#### 12.4.6 算法

采用**K-Means聚类算法**提取主色调，步骤如下：

```
extractColors(imageFile, count):
  1. 读取图片文件，创建Image对象
  2. 调用preprocessImage()将图片缩放到maxWidth
  3. 从Canvas获取像素数据（Uint8ClampedArray）
  4. 采样：每隔N个像素取一个点，减少计算量
  5. 初始化K个聚类中心（随机选取K个像素的RGB值）
  6. 迭代：
     a. 将每个采样像素分配到最近的中心（欧氏距离）
     b. 重新计算每个聚类的中心（各通道平均值）
     c. 若中心变化小于阈值或达到最大迭代次数，终止
  7. 计算每个聚类的像素数量（面积占比）
  8. 按面积降序排列，取前count个颜色
  9. 转换为HEX格式，返回数组
```

**备选方案**：为降低实现复杂度，可使用成熟的前端npm库（如`color-thief`）实现主色提取，其内部采用中位切分法（Median Cut）算法。两种方案的技术路径：

- **K-Means方案**：需自行实现聚类算法，灵活性高但代码量大；
- **Color-Thief方案**：调用成熟库，一行代码即可完成，适合本项目使用AI工具开发的场景。

本项目优先推荐使用`color-thief`库方案，该库已在大量项目中验证稳定性，API简单易用。

#### 12.6.1 程序描述

#### 12.6.6 算法

（1）**ASE格式编码**：
ASE（Adobe Swatch Exchange）为二进制格式。开头4字节为`ASEF`（ASCII），接着4字节为版本号（大端uint32，通常为`0x00000001`），之后是连续的块（block）。每个颜色块包含：块类型（2字节）、块长度（4字节）、颜色名称（以`\0`结尾的UTF-16BE字符串）、颜色模型（4字节，如`“RGB ”`）、颜色数据（RGB各4字节float）、颜色类型（2字节，`0x0002`表示全局色）。

```
exportASE(colors, name):
  buffer = new ArrayBuffer(计算总大小)
  view = new DataView(buffer)
  offset = 0
  
  // 写入文件头
  writeString(view, offset, "ASEF")  // 4字节
  offset += 4
  view.setUint32(offset, 0x00000001, false)  // 版本号，大端
  offset += 4
  view.setUint32(offset, colors.length, false)  // 块数量
  offset += 4
  
  // 写入每个颜色块
  for color in colors:
    rgb = hexToRgb(color)
    // 块类型（0x0001表示颜色条目）
    view.setUint16(offset, 0x0001, false); offset += 2
    // 块长度（稍后回填）
    lengthOffset = offset; offset += 4
    // 颜色名称（UTF-16BE）
    nameBytes = encodeUTF16BE(color)
    for byte in nameBytes: view.setUint8(offset++, byte)
    view.setUint16(offset, 0x0000, false); offset += 2  // 终止符
    // 颜色模型 "RGB "
    writeString(view, offset, "RGB "); offset += 4
    // RGB值（各4字节float，范围0～1）
    view.setFloat32(offset, rgb.r / 255, false); offset += 4
    view.setFloat32(offset, rgb.g / 255, false); offset += 4
    view.setFloat32(offset, rgb.b / 255, false); offset += 4
    // 颜色类型 0x0002 = 全局色
    view.setUint16(offset, 0x0002, false); offset += 2
    // 回填块长度
    blockLength = offset - lengthOffset - 4
    view.setUint32(lengthOffset, blockLength, false)
  
  // 创建Blob并触发下载
  blob = new Blob([buffer], {type: "application/octet-stream"})
  triggerDownload(blob, `${name}.ase`)
```

（2）**CSS导出**：
```
exportCSS(colors):
  css = ":root {\n"
  labels = ["--primary", "--secondary", "--accent", "--neutral", "--background", "--text"]
  for i in range(colors.length):
    css += `  ${labels[i]}: ${colors[i]};\n`
  css += "}"
  
  blob = new Blob([css], {type: "text/css"})
  triggerDownload(blob, "colors.css")
```

（3）**PNG导出**：
```
exportPNG(colors):
  canvas = createCanvas(800, 100)
  ctx = canvas.getContext("2d")
  width = 800 / colors.length
  for i in range(colors.length):
    ctx.fillStyle = colors[i]
    ctx.fillRect(i * width, 0, width, 100)
  
  canvas.toBlob(blob => triggerDownload(blob, "palette.png"))
```

```
class StorageService:
  constructor():
    this.db = null
    this.initDB()
  
  initDB():
    request = indexedDB.open("ColorPaletteDB", 1)
    request.onupgradeneeded = (event) => {
      db = event.target.result
      store = db.createObjectStore("history", {keyPath: "id"})
      store.createIndex("createdAt", "createdAt")
    }
    request.onsuccess = (event) => this.db = event.target.result
  
  savePalette(colors, source, sourceDetail):
    if not this.db: return false
    record = {
      id: generateUUID(),
      colors: colors,
      source: source,
      sourceDetail: sourceDetail,
      createdAt: Date.now(),
      tags: []
    }
    transaction = this.db.transaction(["history"], "readwrite")
    store = transaction.objectStore("history")
    store.add(record)
    
    // 检查记录数，超过50条则删除最早记录
    this.enforceLimit(50)
    return true
```

#### 12.8.1 程序描述

#### 12.8.6 算法

（1）**Canvas绘制色轮**：
色轮是一个360°的HSL色相环。绘制时遍历每个像素，计算其相对于圆心的角度，转换为色相值，固定饱和度100%、明度50%，然后填充。HSL配色规则中常用饱和度100%、明度50%作为标准色相环。

```
renderWheel():
  ctx = canvas.getContext("2d")
  width = canvas.width, height = canvas.height
  centerX = width / 2, centerY = height / 2
  radius = Math.min(width, height) / 2 - 5
  
  for x in range(width):
    for y in range(height):
      dx = x - centerX, dy = y - centerY
      distance = Math.sqrt(dx*dx + dy*dy)
      if distance <= radius:
        angle = Math.atan2(dy, dx) * 180 / Math.PI
        hue = (angle + 90 + 360) % 360  // 调整起始角度
        color = `hsl(${hue}, 100%, 50%)`
        ctx.fillStyle = color
        ctx.fillRect(x, y, 1, 1)
```

（2）**拖拽选色算法**：
```
handleDrag(e):
  rect = canvas.getBoundingClientRect()
  x = e.clientX - rect.left
  y = e.clientY - rect.top
  dx = x - centerX, dy = y - centerY
  
  // 检查是否在色轮内
  distance = Math.sqrt(dx*dx + dy*dy)
  if distance > radius: return
  
  // 计算色相角度
  angle = Math.atan2(dy, dx) * 180 / Math.PI
  hue = (angle + 90 + 360) % 360
  
  // 转换为HEX并触发事件
  hex = hslToHex(hue, 100, 50)
  emit("color:changed", {hex, hue})
```

（3）**性能优化**：
- 色轮只在初始化时绘制一次，拖拽时不重绘；
- 拖拽时只需更新指示器位置（一个圆形标记）而非重绘整个色轮；
- 可使用`OffscreenCanvas`在Worker中绘制色轮，避免阻塞UI。

#### 12.8.9 存储分配

- Canvas像素缓冲区：`width × height × 4`字节（例如200×200时约160KB）；
- 色轮绘制为一次性操作，绘制完成后像素数据保留在Canvas中。

#### 12.8.11 限制条件

- 依赖Canvas API；
- 色轮尺寸固定（建议200×200至300×300像素）；
- 拖拽时需要监听`mousemove`和`mouseup`事件。
