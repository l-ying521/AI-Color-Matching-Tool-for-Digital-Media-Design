## 1 引言

### 1.1 编写目的
本软件需求规格说明书的编制目的是：明确“面向数字媒体设计的AI智能配色辅助工具”的功能需求、性能需求、界面需求及运行环境约束，为项目的后续设计、编码实现和测试验收提供统一的依据。

本说明书的预期读者包括：
- 项目开发者（由AI编程工具Trae辅助生成代码）；
- 项目指导教师（评审功能完整性与规范性）；
- 测试人员（用于功能验收与Bug定位）；
- 课程答辩评审人员（用于了解项目范围与实现深度）。

### 1.2 项目背景
**a. 待开发的软件系统的名称**：面向数字媒体设计的AI智能配色辅助工具（简称“AI配色助手”）。

**b. 项目任务相关方**：
- 任务提出者：本软件为数字媒体技术专业课程设计/毕业设计项目；
- 开发者：大二学生开发者，通过AI编程工具Trae实现代码生成；
- 最终用户：数字媒体设计师（包括UI设计师、平面设计师、视频包装师、插画师等）、设计初学者及有快速配色需求的内容创作者；
- 运行环境：现代浏览器（Chrome 90+、Edge 90+、Firefox 88+、Safari 14+），无需后端服务器部署。

**c. 系统间关系**：本软件为一款独立的、自包含的Web应用程序，不依赖于特定的外部系统，所有核心功能均在浏览器端完成。系统可读取本地图片文件作为输入，并支持色卡导出为外部文件格式（.ase、.css、.png）以供其他设计软件使用。

### 1.3 术语定义与缩略语

| 术语/缩略语 | 定义说明 |
| :--- | :--- |
| **HEX** | 十六进制颜色码（如 #FF6B6B），Web设计中最常用的颜色表示格式 |
| **RGB** | 红绿蓝三通道颜色模型，每个通道取值范围0～255 |
| **HSL / HSV** | 色相-饱和度-明度（值）颜色模型，更符合人类色彩感知方式 |
| **LLM** | 大语言模型（Large Language Model），本项目中用于解析用户的自然语言风格描述 |
| **K-Means** | K均值聚类算法，用于从图片中提取主色调 |
| **色轮规则** | 基于色相环角度关系的配色规则，如互补色（180°）、邻近色（±30°）、三角色（±120°）等 |
| **ASE** | Adobe Swatch Exchange格式，用于在Adobe系列软件中导入色板 |
| **CSS** | 层叠样式表，本项目中指导出可直接用于前端开发的CSS变量代码 |
| **SRS** | 软件需求规格说明书（Software Requirements Specification） |
| **Trae** | 字节跳动推出的AI原生IDE产品，支持通过自然语言描述生成和修改代码 |

## 2 任务概述

### 2.1 项目目标
本项目旨在开发一款面向数字媒体设计师的AI智能配色辅助工具。软件的开发意图是：解决设计师在日常工作中配色耗时长、缺乏色彩理论基础、难以将抽象的风格描述转化为具体色值等痛点问题。系统将提供多源配色生成能力（图片取色、色轮规则生成、随机生成），并通过接入大语言模型实现自然语言驱动的主题配色生成，最终实现从灵感获取到色板导出的全流程辅助。

本软件为一款完全独立的Web应用，全部功能自包含，不依赖于外部系统的接口调用（LLM API调用除外）。用户通过浏览器打开即可使用所有功能，无需安装或登录。

### 2.2 用户特点
本软件的最终用户群体及特征如下：

| 用户类别 | 教育水平 | 技术专长 | 使用频度 | 设计约束 |
| :--- | :--- | :--- | :--- | :--- |
| 专业设计师（UI/平面/视频/插画） | 设计类专业本科及以上 | 熟悉设计软件操作，了解基本色彩理论 | 高频（日常工作使用） | 需要高效的交互体验、专业的色板导出格式 |
| 设计初学者/学生 | 高中至本科在读 | 对色彩搭配缺乏系统认知 | 中频（学习练习使用） | 需要直观易用、学习曲线平缓 |
| 内容创作者/运营人员 | 不限 | 非设计专业，缺少配色技能 | 低频（临时性配色需求） | 需要快速获得可用的配色方案，无需专业知识 |

这些用户特点对软件设计形成以下约束：界面设计应简洁直观，无需专业色彩理论知识即可操作；功能入口应清晰可见，主要操作流程不超过3步；配色生成结果应提供实时预览效果。

### 2.3 假定和约束
本项目的开发工作基于以下假定和约束：

**假定条件：**
1. 用户使用的浏览器支持现代Web API（如FileReader、Canvas、Clipboard API、IndexedDB）；
2. 用户具有基本的计算机操作能力，能够完成文件上传、复制粘贴等基础操作；
3. 对于AI主题化生成功能，假定用户能够通过自然语言清晰描述所需的配色风格；
4. 假定用户上传的图片格式为常见的JPG、PNG或WebP格式，文件大小不超过10MB。

**约束条件：**
1. **经费限制**：本项目为零经费的课程设计项目，只能使用免费或开源的技术资源；
2. **开发期限**：项目须在16周教学周期内完成（含需求分析、设计、编码、测试和文档撰写各阶段）；
3. **技术限制**：项目由大二学生使用AI编程工具Trae完成，代码实现能力有限，应优先选择成熟稳定的技术方案，避免复杂的自研算法；
4. **部署限制**：项目交付物为可在浏览器本地运行的静态网页，不涉及服务器部署与运维；
5. **设备限制**：系统面向桌面端浏览器设计，屏幕分辨率适配最低为1366×768。

## 3 需求规定

### 3.1 对功能的规定
本软件的核心功能分为四大模块：多源配色生成模块、AI主题化生成模块、实用辅助功能模块和数据管理模块。各功能以IPO表（输入-处理-输出）形式逐项说明如下。

#### 3.1.1 图片取色功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户通过点击上传按钮或拖拽方式提交一张图片文件（支持JPG、PNG、WebP格式，文件≤10MB） |
| **处理** | 系统读取图片像素数据，使用K-Means聚类算法提取3～8个主色；用户可通过滑块调整提取颜色数量；系统按色彩占比排序返回主色列表 |
| **输出** | 生成一个由多个主色组成的配色方案（色板），每个颜色以HEX格式显示，并可单击复制色值 |

**补充说明**：图片取色功能可复用成熟的npm库如`color-thief`实现前端主色提取，降低技术实现难度。如遇到处理时间过长的情况，系统应显示“处理中”加载状态。

#### 3.1.2 色轮规则生成功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户通过色轮组件选择一个基色（HEX格式），并从下拉列表中选择一种色彩和谐规则（互补色、邻近色、三角色、矩形色、单色渐变） |
| **处理** | 系统基于HSL色彩空间进行运算：根据所选规则计算目标色相的偏移角度（互补色180°、邻近色±30°、三角色±120°、矩形色±60°/180°），保持饱和度与明度不变，生成目标颜色并转换为HEX格式 |
| **输出** | 生成一个包含基色和计算所得衍生色的配色方案，每个颜色显示其HEX色值和规则标签 |

**补充说明**：色轮组件是核心交互部件，应以Canvas绘制，支持拖拽选色和实时预览。三角色规则指选择在色轮上相隔120度的三种颜色，矩形色规则指选择相隔60度的四种颜色。

#### 3.1.3 随机创意生成功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户点击“随机生成”按钮；用户可设置饱和度范围（0%～100%）和明度范围（0%～100%）的约束条件 |
| **处理** | 系统在用户设定的饱和度和明度约束范围内随机生成一个或多个色相值，组合形成配色方案；每次点击生成一组全新的配色 |
| **输出** | 生成一个随机配色方案，显示各颜色的HEX值；用户可锁定满意的颜色继续生成其他颜色 |

**补充说明**：随机生成功能应提供“锁定”机制——用户可点击某个色块将其锁定，后续随机生成时仅替换未锁定的色块，该功能借鉴了Coolors的经典交互设计。

#### 3.1.4 AI主题化生成功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户在文本框中输入自然语言描述的风格词汇或短句（如“赛博朋克霓虹紫”“温柔奶茶色系”“莫兰迪绿”）；点击“生成”按钮提交 |
| **处理** | 系统将用户输入与预设的Prompt模板组合，通过API发送给大语言模型；LLM解析风格描述，返回结构化的色彩倾向描述（色相范围、饱和度特征、明度特征、推荐色值列表）；前端接收后调用色轮规则算法或直接使用LLM推荐的色值生成最终色板 |
| **输出** | 生成一个符合描述风格的配色方案（5～6个颜色），附带风格标签说明（如“该配色以紫红色为主调，搭配青绿色形成高对比，呈现赛博朋克科技感”），显示在方案描述区域 |

**补充说明**：LLM输出格式应为严格合法的JSON数组，系统需加入JSON解析异常处理机制，解析失败时自动回退到色轮规则生成。AI接入优先采用本地部署方案（如Ollama部署开源模型Qwen2.5或Llama3.2），降低依赖和成本；备用方案为调用免费LLM API（如通义千问、DeepSeek）。

#### 3.1.5 配色方案预览功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 系统当前激活的配色方案（通过上述任一方式生成） |
| **处理** | 系统将配色方案中的各颜色绑定为CSS变量，动态应用到预设的预览模板上；预览模板包括UI卡片示例、插画线稿示例、海报布局示例；系统实时更新预览区域的渲染效果 |
| **输出** | 预览区域内实时显示配色方案在实际设计场景中的应用效果 |

**补充说明**：预览功能是设计师评估配色实用性的关键环节。预览模板应覆盖数字媒体设计的典型场景，帮助用户直观感受配色在实际作品中的效果。

#### 3.1.6 色板微调功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户点击配色方案中的某个色块，弹出HSV调节面板（或下方展开滑块面板），拖动滑块调节色相（0°～360°）、饱和度（0%～100%）、明度（0%～100%） |
| **处理** | 系统实时更新该颜色的HEX值，并联动刷新预览区域；可单独调节某一颜色，也可同时微调多个颜色 |
| **输出** | 微调后的配色方案，色值更新，预览同步刷新 |

#### 3.1.7 色卡导出功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户点击“导出”按钮，从下拉列表中选择导出格式（.ase / .css / .png）；可指定导出的配色方案（当前方案或历史方案） |
| **处理** | .ase导出：将配色方案的各颜色按Adobe Swatch Exchange格式规范编码并生成文件；.css导出：将配色方案各颜色生成为CSS :root变量声明代码；.png导出：将配色方案的色板生成为横向排列的色块图片 |
| **输出** | 触发浏览器文件下载，导出指定格式的文件 |

**补充说明**：.ase格式的导出需要实现Adobe Swatch Exchange文件格式规范，该格式为二进制文件，包含版本信息、颜色数量和颜色数据等结构。.css导出可直接复制或下载文件。Adobe Color等专业工具均支持ASE文件导出。

#### 3.1.8 配色历史管理功能

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户每生成一组新的配色方案，系统自动将其保存至历史记录；用户在历史面板中查看、点击回看或删除某条记录 |
| **处理** | 系统使用浏览器IndexedDB / LocalStorage存储配色方案数据（包括各颜色的HEX值、生成时间、生成方式标签）；历史记录最多保存50条，超出时自动清理最早记录 |
| **输出** | 历史面板以卡片列表形式展示保存的配色方案，点击可恢复为当前激活方案 |

#### 3.1.9 软件支持终端数
本软件为纯前端单用户Web应用，每个浏览器标签页即为一个独立的运行实例，无需支持多用户并发操作。软件应支持同时打开多个浏览器标签页使用，各标签页的数据互不影响。

### 3.2 对性能的规定

#### 3.2.1 精度
- 颜色值的HEX格式必须输出6位十六进制数（如#FF6B6B），RGB各通道数值精度为整数（0～255），HSL/HSV色相精度为整数角度值（0～360），饱和度和明度为整数百分比（0%～100%）；
- 图片取色的K-Means聚类应提取准确的主色调，主色数量误差范围不超过±1个（相对用户指定数量）；
- 色轮规则生成的色相差值精度为±1°。

#### 3.2.2 时间特性要求

| 操作类型 | 响应时间要求 | 说明 |
| :--- | :--- | :--- |
| 界面交互响应 | ≤100ms | 按钮点击、滑块拖动等基础UI交互的视觉反馈 |
| 色轮规则生成 | ≤200ms | 从基色选择到衍生色计算显示完成 |
| 图片上传与预览 | ≤1s | 从图片加载到界面显示缩略图 |
| 图片主色提取 | ≤3s | 从点击提取按钮到色板显示完成（含K-Means聚类计算） |
| AI主题化生成 | ≤8s | 从提交风格描述到配色方案显示完成（含LLM API网络请求） |
| 预览模板渲染 | ≤200ms | 配色方案切换后预览区域更新完成 |

**补充说明**：图片主色提取的时间要求与图片分辨率正相关，前端处理时应将图片压缩至合理尺寸（如最大宽度800px）以控制计算量。AI主题化生成的响应时间受LLM API服务状态影响，实际时间可能有所波动，应显示等待状态提示。

#### 3.2.3 灵活性
本软件应具备以下灵活性，以适应需求变化：

- **操作方式的变化**：支持键盘快捷键操作（如空格键触发随机生成、Ctrl/Cmd+C复制色值），以满足高级用户的使用习惯；
- **运行环境的变化**：应兼容主流浏览器的近期版本（Chrome 90+、Edge 90+、Firefox 88+、Safari 14+），核心功能在无网络连接时（除AI生成功能外）仍可使用；
- **界面适配的变化**：采用响应式设计，在1366×768及以上分辨率的桌面端屏幕正常显示，布局不产生错位或截断；
- **配色规则的变化**：色轮规则模块应预留扩展接口，便于后续增加新的和谐规则（如分割互补色、双互补色等）；
- **导出格式的变化**：导出模块应设计为可扩展架构，便于后续增加新的导出格式（如SCSS、SVG等）；
- **AI模型的变化**：LLM调用模块应预留接口配置能力，便于在不同模型提供商之间切换。

### 3.3 输入输出要求

#### 3.3.1 输入数据类型与格式

| 输入项 | 数据类型 | 媒体/方式 | 格式/范围 | 精度 |
| :--- | :--- | :--- | :--- | :--- |
| 图片文件 | 图像 | 本地文件上传/拖拽 | JPG、PNG、WebP | 文件≤10MB |
| 风格描述文本 | 字符串 | 文本框输入 | 1～100个字符，支持中英文 | — |
| 基色选择 | HEX颜色值 | 色轮组件/取色器 | #000000～#FFFFFF | 6位十六进制 |
| 饱和度约束 | 数值 | 滑块 | 0%～100% | 整数百分比 |
| 明度约束 | 数值 | 滑块 | 0%～100% | 整数百分比 |

#### 3.3.2 输出数据类型与格式

| 输出项 | 数据类型 | 媒体/方式 | 格式 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| 配色方案 | 颜色数组 | 界面显示 | 5～6个HEX色块 | #FF6B6B, #4ECDC4, #FFE66D |
| ASE色板文件 | 二进制文件 | 文件下载 | .ase格式 | — |
| CSS变量代码 | 文本 | 界面展示/文件下载 | .css文件或:root代码块 | `--primary: #FF6B6B;` |
| PNG色板图片 | 图像文件 | 文件下载 | .png格式 | 横向排列的色块图片 |
| 预览效果 | 图形显示 | 界面渲染 | CSS变量动态渲染 | 实时显示 |
| 风格标签说明 | 文本 | 界面显示 | 1～2句自然语言描述 | “赛博朋克霓虹紫” |

对硬拷贝报告（状态输出及异常输出）的描述：
- 正常结果输出：配色方案色板、预览区域渲染效果、操作成功提示（短暂显示后自动消失）；
- 状态输出：图片上传加载动画、AI生成等待提示、处理中进度指示；
- 异常输出：错误提示（红色文字或提示条），包括“上传失败：文件格式不支持”“图片处理超时，请重试”“AI生成失败：请检查网络连接”等。

### 3.4 数据管理能力要求
本软件为纯前端应用，所有数据存储在浏览器本地，不涉及后端数据库。数据管理要求如下：

| 数据项 | 存储方式 | 预估规模 | 存储要求 |
| :--- | :--- | :--- | :--- |
| 配色历史记录 | IndexedDB / LocalStorage | 最多50条配色方案 | 每条记录包含5～6个HEX色值、生成时间戳、生成方式标签，总存储量≤50KB |
| 用户偏好设置 | LocalStorage | 若干项设置项 | 包括默认饱和度范围、默认明度范围、上次使用的导出格式等，总存储量≤5KB |
| 应用状态 | 内存（运行时） | 单会话 | 当前激活配色方案、预览模板状态等，会话结束后不持久化 |

系统需实现历史记录的自动清理策略：当记录数超过50条时，自动删除最早生成的记录，确保存储空间在合理范围内。数据存储无需考虑多用户隔离，但需保证同一浏览器的多次访问数据可持久化保存。

### 3.5 故障处理要求

#### 3.5.1 可能的软件故障及处理

| 故障类型 | 可能后果 | 处理要求 |
| :--- | :--- | :--- |
| 图片上传格式不支持 | 无法提取主色 | 前端校验文件扩展名和MIME类型，不符合时提示“请上传JPG、PNG或WebP格式图片”，不进入后续处理流程 |
| 图片文件过大 | 处理时间过长或浏览器卡顿 | 前端校验文件大小，超过10MB时提示“文件过大，请选择10MB以下的图片” |
| 图片主色提取超时 | 用户体验差 | 设置3秒超时阈值，超时后中断计算并提示“处理超时，请尝试压缩图片后重试” |
| LLM API调用失败 | AI主题化生成不可用 | 捕获网络错误和HTTP非2xx响应，提示“AI生成暂时不可用，请稍后重试”，并自动回退到色轮规则生成备用方案 |
| JSON解析异常 | AI返回格式错误导致配色生成失败 | 捕获JSON.parse异常，回退到色轮规则生成，并向用户说明“AI生成失败，已为您切换至规则配色” |
| 浏览器不支持IndexedDB | 历史记录功能不可用 | 检测API可用性，不支持时回退到LocalStorage，仍不支持则禁用历史记录功能并给出提示 |
| 网络断开 | AI主题化生成功能不可用 | 其他离线功能（图片取色、色轮规则、随机生成、微调、导出）仍可正常使用 |

#### 3.5.2 故障恢复
- 所有用户输入数据应在操作前保存在内存中，故障发生时确保已生成的数据不丢失；
- 历史记录数据应持久化保存，浏览器关闭后重新打开可恢复历史记录；
- AI生成失败时，系统应保留用户输入的风格描述文本，便于用户稍后重试或手动调整。

### 3.6 其他专门要求

#### 3.6.1 易用性要求
- 界面布局清晰，主要功能区（生成方式切换、色板展示、预览区域、微调面板）分区明确；
- 常用操作提供快捷入口，如色块单击复制色值、空格键快速随机生成；
- 所有交互元素（按钮、滑块、输入框）应有清晰的视觉提示和悬浮说明文字；
- 首次使用时可显示简单的功能引导提示，但不应干扰正常操作。

#### 3.6.2 可靠性要求
- 配色方案生成算法应保证在相同输入条件下输出一致的结果（随机生成功能除外）；
- 导出文件格式应符合对应规范，确保在目标软件（Adobe系列、代码编辑器）中可正常打开；
- 系统应兼容主流浏览器的近期版本，核心功能在各浏览器中表现一致。

#### 3.6.3 可维护性要求
- 前端代码应模块化组织，核心算法（色彩转换、规则生成）封装为独立模块，便于后续维护和扩展；
- 代码注释应充分，便于其他开发者理解（尤其在使用AI生成代码的情况下，需确保代码可读性）；
- 配置文件（如LLM API端点、Prompt模板）应集中管理，便于切换模型或调整参数。

#### 3.6.4 可移植性要求
- 项目交付物为静态HTML、CSS、JavaScript文件集合，无需任何构建步骤即可在任意静态Web服务器或本地文件系统中运行；
- 项目不依赖特定操作系统，在Windows、macOS、Linux上均可通过浏览器正常使用。

## 4 运行环境规定

### 4.1 设备

#### 4.1.1 硬件设备
本软件为Web应用，对硬件要求较低，最低配置如下：
- 处理器：双核CPU，主频≥1.5GHz；
- 内存：≥4GB RAM；
- 显示器：分辨率≥1366×768；
- 输入设备：鼠标+键盘（触摸屏可选）。

#### 4.1.2 支持软件

| 软件类别 | 要求 |
| :--- | :--- |
| 操作系统 | Windows 10+ / macOS 11+ / Ubuntu 20.04+ / 其他支持现代浏览器的系统 |
| 浏览器 | Chrome 90+ / Edge 90+ / Firefox 88+ / Safari 14+ |
| 其他 | 无需安装任何浏览器插件或扩展；AI功能需要网络连接 |

### 4.2 接口
本软件为独立Web应用，内部接口包括：
- 前端各组件之间的通信接口（事件总线/状态管理）；
- 与LLM API的外部HTTP接口（发送风格描述，接收配色推荐结果）；
- 与浏览器存储API的接口（IndexedDB / LocalStorage）；
- 与浏览器文件API的接口（FileReader读取图片，Blob/URL触发文件下载）。

**外部API说明**：AI主题化生成功能需要调用大语言模型API。建议优先采用本地部署方案（如Ollama + 开源模型），避免外部API依赖；如使用云端API，应选用提供免费额度的服务（如通义千问、DeepSeek等），并做好调用失败时的降级处理。
